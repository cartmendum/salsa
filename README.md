## 🚀 The Vision
**Fast-ASDLC** is not just a collection of prompts; it is a complete engineering methodology designed for the AI era. It automates **80-90% of routine SDLC tasks**—from requirements gathering to DevOps manifests—while maintaining 100% architectural integrity through strict adherence to DDD, Hexagonal Architecture, and SOLID principles.

### Why Fast-ASDLC?


| Feature | Legacy SDLC | Fast-ASDLC |
| :--- | :--- | :--- |
| **Time to Market** | Weeks/Months | **Days/Hours** |
| **Test Coverage** | 20-60% (manual) | **90-100% (automated)** |
| **Documentation** | Outdated/Missing | **Always Sync (Everything-as-Code)** |
| **Architectural Drift**| Common | **Impossible (Spec-Driven)** |
| **Costs** | High (Human-only) | **Zero Token Costs (Local AI)** |

## 🏗 Key Architectural Pillars
- **Agentic Roles:** Specialized agents for Analysis, Architecture, Dev, QA, and SecOps.
- **Everything-as-Code:** All artifacts (C4 models, Threat Models, Runbooks) are Markdown/Mermaid.js to optimize context window.
- **Spec-Driven Development:** No code is generated without a verified architectural specification.
- **Data Sovereignty:** Built for open weights. Deploy on your private infrastructure to eliminate data leaks.
- **Meta-Agent:** A dedicated agent that continuously tunes the skills and workflows of other agents.

## 🛠 Repository Structure
Use this repository as a **Template** to start your new project:
- `.agents/` — Rules, Workflows, and Skills for your AI team.
- `.backlog/` — Intentions and feature definitions.
- `src/` — Hexagonal-layered source code (Domain, Application, Infrastructure).
- `tests/` — Automated test suite (Unit, Integration, Performance).
- `infra/` — Infrastructure-as-Code (Docker, K8s, Terraform) created by DevOps Agent.
- `docs/` — Live artifacts (DDD, C4 Models, Security, Specs), SRE & Ops documentation.
